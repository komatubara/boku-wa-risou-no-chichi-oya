# ゲーム仕様書: ぼくはりそうのちちおや

## 1. 基本情報
* **タイトル:** ぼくはりそうのちちおや
* **ジャンル:** Reignsライク・カードスワイプ型ギャグシミュレーション
* **ターゲット:** 手軽にブラックジョークや風刺を楽しみたいプレイヤー
* **開発エンジン:** Godot Engine 4.x
* **コアコンセプト:** 「息子のため」という名目で親が暴走し、結果的に息子の自主性を奪ってダメ人間にしていく過程を笑うバカゲー。

## 2. コアループ（ゲームシステム）
プレイヤーは父親となり、次々と発生する息子のイベントに対して「Reigns風の2択スワイプ」で介入していく。

1. **イベント発生:** 画面中央にイベントカードが表示される。
2. **選択（スワイプ）:**
   * 左スワイプ: まともな親の対応（見守る、諭すなど）
   * 右スワイプ: りそうのちちおや（モンペ）の対応（金でもみ消す、権力行使、過剰防衛など）
   * ※スワイプ中、画面上部の4つのパラメーターアイコン上に影響度（大小の丸）をプレビュー表示する。
3. **結果表示:** 選択確定後、カードが飛び、結果のギャグテキスト（とパラメーター増減）がポップアップ表示される。
4. **フェーズ進行:** 一定枚数のカードをこなすと息子の年齢が上がり、最終的に社会人フェーズでエンディングを迎える。

## 3. パラメーター仕様
以下の4つのパラメーター（0〜100）を管理する。初期値はすべて50。
いずれかが 0 または 100 に達した場合、その時点で特殊なバッドエンドとなる。

| アイコン | パラメーター | キー | 上限/下限到達時の結末 |
|---|---|---|---|
| 🧠 | 自立心 | `independence` | 0 → 究極のパラサイトニートエンド |
| 👑 | ワガママ度 | `ego` | 100 → 親への暴力・犯罪者エンド |
| 💰 | 親の財力 | `wealth` | 0 → 自己破産エンド |
| 👁️ | 世間のヘイト | `hate` | 100 → 警察介入・社会的抹殺エンド |

## 4. フェーズ進行

全フェーズでイベントカードは合計25枚。各フェーズ5枚消化で次フェーズへ進む。
エンドはバッドエンドのみ（ノーマルエンド・グッドエンドは存在しない）。

| フェーズ | カード枚数 | 備考 |
|---|---|---|
| 幼児 | 5枚 | — |
| 小学校 | 5枚 | — |
| 中学校 | 5枚 | — |
| 高校 | 5枚 | — |
| 社会人 | 5枚 | 最終フェーズ。最終カードでエンディング分岐 |

## 5. エンディング一覧

エンドはすべてバッドエンド。途中でいずれかのパラメーターが限界値に達するか、社会人フェーズ終了時のパラメーター状態で分岐する。

| # | エンド名 | ending_id | 発生条件 | タイトル |
|---|---|---|---|---|
| 1 | 🧠 ニートエンド | `end_neet` | `independence` = 0 | りそうのすみか |
| 2 | 👑 犯罪者エンド | `end_criminal` | `ego` = 100 | モンスターの産声 |
| 3 | 💰 自己破産エンド | `end_bankrupt` | `wealth` = 0 | りそうのだいしょう |
| 4 | 👁️ 社会的抹殺エンド | `end_cancelled` | `hate` = 100 | 炎上家族 |

エンディングデータは `data/endings.json` に格納。

### 社会人フェーズクリア時の分岐ルール

25枚すべてをこなしてもパラメーターが限界値に達しなかった場合、**その時点で限界値に最も近いパラメーター**に応じてエンディングへ分岐する。

| 条件 | 分岐先 |
|---|---|
| `independence` が最低値 | 🧠 ニートエンド |
| `ego` が最高値 | 👑 犯罪者エンド |
| `wealth` が最低値 | 💰 自己破産エンド |
| `hate` が最高値 | 👁️ 社会的抹殺エンド |

同率の場合は上記の優先順位で決定。

## 6. シナリオデータ構造 (JSON形式)
各イベントカードは以下のJSONスキーマに従って定義し、Godot側でパースして使用する（またはCustom Resourceに変換する）。

```json
{
  "card_id": "unique_id",
  "phase": "小学校",
  "character_image": "image_path.png",
  "event_text": "イベントの状況説明文",
  "swipe_left": {
    "action_text": "まともな選択肢の短いテキスト",
    "result_text": "選択後の結果・ギャグ描写",
    "stat_changes": {
      "independence": 10,
      "ego": -5,
      "wealth": 0,
      "hate": 0
    }
  },
  "swipe_right": {
    "action_text": "モンペ選択肢の短いテキスト",
    "result_text": "過干渉によるギャグ描写と息子への悪影響",
    "stat_changes": {
      "independence": -15,
      "ego": 20,
      "wealth": -10,
      "hate": 30
    }
  }
}
```

`stat_changes` の値は -100〜+100。
シナリオデータは `res://data/cards/` 以下に配置する。

### テキストトーン方針
- **じわじわエスカレート系** — 幼児フェーズは比較的穏やか、社会人フェーズに近づくほど被害が大きくなる
- **末路強調スタイル** — 右スワイプの `result_text` は必ず息子・父への具体的な悪影響で締める
- **語り口** — フェーズ・カードに応じて客観ナレーション／息子の声／父の内心を使い分ける

### stat_changes フェーズ別設計指針
右スワイプ1回あたりの変動幅の目安。序盤は小さく、後半ほど大きくする。
※バランス調整済み（v2）。全右スワイプで中学校3枚目にhate=100、高校2枚目にindependence=0到達。

| フェーズ | hate 変動幅（目安） | independence 変動幅（目安） | ego 変動幅（目安） |
|---|---|---|---|
| 幼児 | +2 | 0〜-2 | 0〜+5 |
| 小学校 | +3〜+8 | -2〜-5 | +3〜+5 |
| 中学校 | +5〜+8 | -4〜-5 | +3〜+5 |
| 高校 | 0〜+8 | -4〜-7 | +3〜+8 |
| 社会人 | +7〜+10 | -5〜-9 | +3〜+8 |

## 8. 技術仕様

### 基本設定

| 項目 | 内容 |
|---|---|
| エンジン | Godot Engine 4.x |
| エクスポート | Web（HTML5）/ WebGL 2.0（Compatibilityレンダラー） |
| シナリオデータ | JSON形式（`res://data/cards/`） |
| ピクセルアート生成 | PixelLab MCP |
| 音声生成 | ElevenLabs MCP（出力先: `assets/audio/`） |

### 投稿先プラットフォームと制約

| プラットフォーム | 制約・注意点 |
|---|---|
| **Plicy** | GodotはiframeでのHP埋め込み不可。「大画面表示のみ」設定を使用。エクスポートHTMLを `index.html` にリネームしてzip化して投稿。 |
| **GodotPlayer** | SharedArrayBuffer対応済み。index.htmlリネーム不要。登録時にブラウザサイズを入力。 |

> ⚠️ Godot 4 WebエクスポートはCompatibilityレンダラー（WebGL 2.0）のみ。Safariでは動作不安定なためChromeを推奨。

### 画面解像度設定

| 項目 | 設定値 |
|---|---|
| 基本解像度 | `720 × 1280`（縦型） |
| ストレッチモード | `canvas_items`（ピクセルアートを滲まず拡大縮小） |
| アスペクト比 | `expand`（PCの横長モニターで上下に余白追加、破綻しない） |

### UI・フォントサイズ基準

スマホユーザーも想定し、小画面での可読性を最優先とする。

| 要素 | 最小サイズ | 推奨サイズ |
|---|---|---|
| event_text（状況説明） | 20px | **24px** |
| action_text（選択肢ラベル） | 18px | **22px** |
| result_text（結果テキスト） | 18px | **22px** |
| パラメーター数値 | 16px | **20px** |
| タッチ操作ターゲット（アイコン等） | 44×44px | **60×60px** |

### 入力操作

| 操作 | 対応方法 |
|---|---|
| PCマウスドラッグ | `InputEventMouseMotion` |
| スマホタッチスワイプ | `InputEventScreenDrag` |
| 設定 | Godotの `touch_emulation` を有効化（PC上でタッチ操作をエミュレート） |

## 10. BGM・SE仕様

音声素材はすべてElevenLabs MCPで生成。出力先: `assets/audio/`

### BGM方針

**テイスト:** 表向き明るく・コミカル、しかしフェーズが進むにつれ音が歪み不穏になっていく。
プレイヤーが「あれ、なんか変だな」と気づく頃には手遅れになっている風刺感を音で演出する。

| フェーズ | ファイル名 | 方向性 |
|---|---|---|
| 幼児 | `bgm_infant.mp3` | おもちゃ箱のような無邪気でかわいい音楽 |
| 小学校 | `bgm_elementary.mp3` | 元気でコミカル、少し賑やかな学校っぽさ |
| 中学校 | `bgm_junior_high.mp3` | テンポが上がり、和音がやや不安定になる |
| 高校 | `bgm_high_school.mp3` | 表向き青春っぽいが、ところどころ音が歪む |
| 社会人 | `bgm_adult.mp3` | 穏やかなオフィスBGM風、でも低音に不穏な音が混じる |

### SE一覧

| 役割 | ファイル名 | タイミング | 音のイメージ |
|---|---|---|---|
| カードスワイプ | `se_swipe.mp3` | カードを左右にドラッグしたとき | 紙をめくる・滑らせる音 |
| パラメーター上昇 | `se_param_up.mp3` | パラメーターが増加したとき | 軽やかな上昇音（チーン） |
| パラメーター下降 | `se_param_down.mp3` | パラメーターが減少したとき | 重めの下降音（ズン） |
| 結果テキスト表示 | `se_popup.mp3` | 結果ポップアップが出るとき | ポンと出る軽い効果音 |
| バッドエンド突入 | `se_bad_end.mp3` | いずれかのパラメーターが限界値に達したとき | ドーンという重い演出音 |
