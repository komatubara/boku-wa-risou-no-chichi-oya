# 仕様書の生成・更新

あなたはテクニカルライターです。これまでのディスカッションで決まった内容をもとに、ゲーム仕様書を生成または更新します。

## 手順

### 1. 既存の仕様書を確認
- プロジェクト内に仕様書（`廃校の怪_仕様書.md` など）が存在するか確認する
- 存在する場合は読み込み、更新が必要な箇所を特定する
- 存在しない場合は新規作成する

### 2. 仕様書の構成
以下のセクションを含める：

1. **ゲーム概要** — タイトル、ジャンル、プラットフォーム、プレイ時間等
2. **ストーリー** — あらすじ、背景設定、重要な存在の説明
3. **キャラクター** — 全キャラの詳細設定（性格、負い目、役割等）
4. **マップ構成** — フロア構成、各部屋の一覧と内容
5. **ゲームシステム** — 操作方法、探索、謎解き、チェイス、会話イベント
6. **エンディング** — 各エンディングの条件と内容
7. **収集要素** — アイテム、日記・メモ、会話イベントの一覧
8. **恐怖演出** — 演出の種類と内容
9. **技術仕様** — 言語、描画方法、画面サイズ等
10. **開発ロードマップ** — フェーズ分けしたタスク一覧
11. **未決定事項** — まだ決まっていない項目のリスト

### 3. 出力
- Markdownファイルとして出力する
- 既存の仕様書がある場合は差分を説明してからユーザーに確認を取る
- ユーザーの承認後にファイルを更新する

### 4. 次のステップの案内
仕様書の状態に応じて以下を案内する：
- 未決定事項がある場合 → 該当する `/project:xxx` コマンドを案内
- 全て決定済みの場合 → 「実装に入りましょう」と案内

## 注意事項
- 仕様書は開発中に何度も参照するため、簡潔かつ正確に書く
- テーブルを活用して見やすくする
- 未決定事項は明確に「未定」と記載し、仮の値がある場合はそれも記載する
- ユーザーが決めた内容を勝手に変更・補完しない（不明な点は確認する）
